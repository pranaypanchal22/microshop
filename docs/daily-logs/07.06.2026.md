## What I Learned
- Pydantic schemas and SQLAlchemy models are separate on purpose — schemas
  control what the API exposes, models control what the database stores.
  Never mix them into one class.
- model_dump(exclude_unset=True) only updates fields the client actually sent.
  Without this, a PATCH that only sends price would wipe out all other fields.
- db.refresh(db_product) is required after commit to get auto-generated values
  like id and created_at back into the Python object before returning it.
- Always read the full error traceback from the bottom up — the last line
  tells you the error type, the line above tells you exactly where it happened.
- Never run curl commands from PowerShell — use Ubuntu terminal only.
  PowerShell wraps responses in verbose objects and breaks multi-line commands.
