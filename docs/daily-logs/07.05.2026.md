## What I Learned
- PostgreSQL requires a dedicated application user — never use the postgres
  superuser for your application. Principle of least privilege.
- SQLAlchemy models are Python classes that map directly to database tables.
  The __tablename__ attribute tells SQLAlchemy which table to use.
- Alembic generates migration files automatically by comparing your Python
  models against the actual database schema — you don't write SQL manually.
- alembic upgrade head applies all pending migrations up to the latest one.
  head always means the most recent migration in the chain.
- The .env file must exist BEFORE you write any code that uses secrets.
  If a secret is ever committed to Git, it lives in history forever.
- In WSL2, PostgreSQL does not start automatically — run
  sudo service postgresql start at the beginning of every session.
