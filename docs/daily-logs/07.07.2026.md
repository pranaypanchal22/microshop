## What I Learned
- Redis is an in-memory store — reads are 10-100x faster than PostgreSQL
  because data lives in RAM, not on disk
- Cache-aside pattern: check cache first, on miss query DB and store result,
  on write delete the cache so next read gets fresh data
- SETEX = SET with EXpiry — stores a key with automatic TTL deletion
- Cache invalidation must happen on every write (POST/PUT/DELETE) or users
  see stale data
- TTL is a safety net — even if invalidation has a bug, cache expires
  automatically after N seconds
- redis-cli monitor shows every Redis command in real time —
  essential tool for debugging cache behaviour
- redis-server and redis-tools are separate packages on Ubuntu
